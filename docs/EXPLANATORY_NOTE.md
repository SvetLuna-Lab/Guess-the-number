# Пояснительная записка
## к учебному проекту «Числовая угадайка» (Python)

**Дисциплина:** Программирование (Python)  
**Тема:** Функции, циклы, модуль random, обработка ввода, тестирование  
**Формат:** консольное приложение

---

## 1. Цель работы
Разработать консольную игру «Числовая угадайка», в которой компьютер загадывает случайное число, а пользователь пытается его угадать. В ходе выполнения проекта закрепить навыки работы с функциями, циклами, условными операторами, модулем `random`, а также реализовать защиту от некорректного ввода и выполнить базовое тестирование.

---

## 2. Задачи работы
1. Подключить модуль `random` и реализовать генерацию случайного числа.
2. Организовать взаимодействие с пользователем (приветствие, подсказки, завершение программы).
3. Реализовать валидацию пользовательского ввода (защита от ошибочных данных).
4. Реализовать основной цикл угадывания с подсказками «больше/меньше».
5. Добавить:
   - подсчёт попыток;
   - возможность повторной игры;
   - возможность задавать правую границу диапазона (1..n).
6. Оформить проект в виде репозитория с документацией.
7. Написать автотесты и настроить CI (GitHub Actions) для автоматической проверки.

## 4. Описание программы
### 4.1. Функциональные возможности
Программа:
- запрашивает правую границу диапазона `n` (натуральное число);
- генерирует случайное число `secret` в диапазоне от 1 до n;
- принимает ввод пользователя и проверяет корректность;
- выводит подсказки:
  - «меньше загаданного»,
  - «больше загаданного»,
  - «вы угадали»;
- считает количество попыток (засчитываются только корректные числа);
- предлагает сыграть повторно;
- корректно завершает работу с прощанием.

### 4.2. Валидация ввода
Ввод считается корректным, если:
- введена строка, представляющая целое число;
- число лежит в диапазоне `1..n`.

При некорректном вводе выводится сообщение:
`А может быть все-таки введем целое число от 1 до N?`

---

## 5. Алгоритм работы (блок-схема словами)
1. Вывести приветствие.
2. Запросить правую границу `n`:
   - пока ввод неверный — повторять запрос.
3. Сгенерировать `secret = randint(1, n)`.
4. Запустить цикл угадывания:
   - запросить число пользователя;
   - если ввод неверный — вывести предупреждение и продолжить цикл;
   - если ввод верный:
     - увеличить счётчик попыток;
     - сравнить с `secret` и вывести подсказку;
     - при равенстве завершить раунд.
5. Вывести количество попыток.
6. Запросить «сыграть ещё раз?»:
   - если да — перейти к шагу 2;
   - если нет — вывести прощание и завершить программу.

## 7. Листинг ключевых частей программы
### 7.1. Валидация ввода
Используется функция `is_valid(text, right)`:

- возвращает `True`, если `text` — целое число и `1 <= int(text) <= right`;
- иначе возвращает `False`.

### 7.2. Основной цикл
Функция `run_game()`:
- запускает игру;
- организует повторные раунды;
- завершает программу с прощанием.

---

## 8. Тестирование
Для проверки корректности реализованы автотесты `pytest`.

Проверяются:
- корректность `is_valid()` на разных вводах;
- корректность подсчёта попыток;
- корректная обработка правой границы диапазона `n`;
- корректная обработка ответа на повторную игру.

Запуск тестов:
```bash
python -m pip install -e .[dev]
pytest
```
Также настроен GitHub Actions workflow CI, который запускает тесты автоматически при push и pull request.
